strong과 b

strong태그는 사실 그 자체만으로는 글자를 진하게 하라는 의미는 없습니다.
왜냐하면 HTML과 CSS가 명확히 분리되면서 HTML은 오직 문서의 논리적 구조만을 지정하는 역할만을 맡게 되었거든요
즉, strong태그로 표기해 놓은 건 '이 부분의 글자를 강조해 줘' 라고 전달만 할 뿐, 어떻게 강조할지에 대해서는 HTML은 신경쓰지 않습니다
HTML이 strong이라고 표기해 놓은 부분의 글자를 어떻게 강조할지는 전적으로 CSS 혹은 각 브라우저의 몫입니다
(물론 대부분의 브라우저는 strong의 처리를 진하게 하고 끝냅니다)

b태그도 마찬가지입니다
b태그로 표기해 놓은 건 '이 부분의 글자를 진하게 해 줘' 라고 전달만 할 뿐, 얼마나 진하게 할지에 대해서는 HTML은 신경쓰지 않습니다
HTML이 b라고 표기해 놓은 부분의 글자를 얼마나 진하게 할 지는 전적으로 CSS 혹은 각 브라우저의 몫입니다
